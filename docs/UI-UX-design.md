## 1. 설계 원칙

### 1.1 핵심 원칙 3가지

| 원칙 | 설명 | 적용 |
|------|------|------|
| **30초 원칙** | 가입부터 핵심 가치 경험까지 30초 이내 | 옷장을 채우는 것이 아닌, 경험을 먼저 준다 |
| **1-Tap 접근** | 모든 핵심 기능은 홈에서 1탭 이내 | 룩 재현, 옷장, 코디 기록 모두 탭바로 접근 |
| **비교 중심 UI** | 룩 재현의 핵심은 직관적 비교 | 매칭 결과에서 매칭/갭 아이템을 명확히 구분 |

### 1.2 UX 톤 & 무드

- **긍정적 언어:** "없는 아이템" → "이 아이템이 있으면 완벽해요!"
- **가벼운 톤:** 패션은 즐거운 것. 무겁거나 사무적이지 않게.
- **성취감 피드백:** 매칭 성공 시 confetti 효과, 옷장 확장 시 축하 메시지
- **미니멀 UI:** 정보 과부하 없이, 한 화면에 하나의 핵심 액션

### 1.3 디자인 무드 (v4.9)

| 항목 | 값 |
|------|---|
| 테마 | 글래스모피즘 · 소프트 그라디언트 · 에디토리얼 |
| 브랜드 컬러 | Indigo (#4F46E5) |
| 페이지 타이틀 | 영문 Lobster 32px |
| 본문 | Noto Sans KR |
| 카드 | 반투명 글래스 (backdrop-filter: blur) |
| 배경 | 소프트 라벤더~스카이 그라디언트 (bgSoft) |
| 탭바 | 화이트 글래스 알약, 아이콘 전용 |

## 2. 정보 구조 (Information Architecture)

### 2.1 네비게이션: 플로팅 알약 탭바 (4탭)

```
         ┌──────────────────────────────┐
         │  [🏠]   [📦]   [🔍]   [👤]  │
         │  Home  Closet  Match    My   │
         └──────────────────────────────┘
```

| 탭 | 아이콘 | Lobster 타이틀 | 기능 |
|----|--------|---------------|------|
| Home | 집 (outline/filled) | Aura | 대시보드, 통계, 퀵 액션, 최근 매칭 |
| Closet | 4분할 그리드 | Closet | 카테고리 필터 + Masonry 그리드 + FAB |
| Match | 스캔 뷰파인더 | Match | 룩 매칭 업로드 + 결과 |
| My | 사람 | My | 플랜 배너, 코디 캘린더, 설정 메뉴 |

**탭바 디자인 스펙 (v4.9):**
- 배경: `rgba(255,255,255,0.92)` + blur(24px)
- 라운딩: 22px (외부 컨테이너)
- 탭 아이템: 44×44px, radius 14px, 아이콘만 (라벨 없음)
- 활성 탭: 배경 `rgba(79,70,229,0.1)` + 아이콘 `#4F46E5` (filled)
- 비활성 탭: 투명 배경 + 아이콘 `#999` (outline)
- 그림자: `0 2px 20px rgba(0,0,0,0.08)`

**설계 근거:**
- 4탭 구성 — 5탭 이상은 터치 타겟이 좁아지고 인지 부하 증가
- "Match"를 독립 탭으로 배치 — 핵심 차별점이므로 최대 노출
- 라벨 제거 — 아이콘만으로 직관적, 탭바가 더 컴팩트하고 세련됨

## 3. 사용자 흐름도 (User Flows)

### 3.1 플로우 A: 온보딩 (가입 → 첫 경험)

**목표:** 30초 안에 핵심 가치 경험

```
앱 실행
  │
  ▼
#1. 스플래시 A (인디고 배경, Lobster 72px 화이트 로고)
  │  자동 1.5초
  ▼
#2. 스플래시 B (소프트 배경, Lobster 72px 인디고 로고)
  │  자동 1초
  ▼
#3. 온보딩 1 (인디고 배경)
  │  중앙: 옷걸이 동심원 아이콘
  │  "나만의 옷장을 AI로 관리하세요"
  │  "새로운 스타일을 발견해보세요"
  │  하단: [시작하기] (글래스 버튼)
  │  상단 우측: "건너뛰기" (→ #6 로그인)
  │
  ▼ (스와이프 또는 버튼)
#4. 온보딩 2 (인디고 배경)
  │  ← 뒤로가기 (좌상단)
  │  중앙: 카메라 동심원 아이콘
  │  "사진 한 장이면 충분해요"
  │  "AI가 옷을 자동으로 분석합니다"
  │  하단: [다음] (글래스 버튼)
  │
  ▼
#5. 온보딩 3 (인디고 배경)
  │  ← 뒤로가기 (좌상단)
  │  중앙: 스캔 동심원 아이콘
  │  "인플루언서 룩을 내 옷으로"
  │  "내 옷장에서 완벽한 매칭을 찾아드려요"
  │  하단: [시작하기] (글래스 버튼)
  │
  ▼
#6. 로그인 (소프트 배경)
  │  ← 뒤로가기 (좌상단)
  │  중앙 상단: Lobster 56px "Aura" (인디고) + "STYLE REIMAGINED"
  │  하단:
  │  ├── [카카오로 시작하기] (#FEE500 배경, 텍스트 #1A1A1A)
  │  ├── [Apple로 시작하기] (#000 배경, 텍스트 #fff)
  │  └── 법적 고지 (11px, mute)
  │
  ▼ (로그인 성공)
#7. 홈 (Home 탭)
```

**핵심 UX 포인트:**
- 온보딩 슬라이드는 **건너뛰기 가능** (우상단, 어떤 화면에서든 → 로그인)
- 온보딩 2~3에 **뒤로가기** 지원 (이전 슬라이드로)
- 카카오 로그인 버튼을 **가장 위에** 배치 (한국 사용자 대부분)
- 온보딩 아이콘은 **동심원 원형** — 외부 링 120px, 중간 90px, 내부 50px
- 모든 온보딩 텍스트 **중앙 정렬**, 배경 인디고 그라디언트

### 3.2 플로우 B: 인플루언서 룩 재현 (핵심 흐름)

**목표:** 레퍼런스 이미지 → 매칭 결과까지 10초 이내

```
#9. 매칭 업로드 (Match 탭)
  │  헤더: "Match" Lobster 32px
  │  서브: "인플루언서 룩을 내 옷으로 재현해보세요"
  │  
  │  [사진 업로드 영역] (글래스 카드, 점선 테두리, 중앙 카메라 아이콘)
  │  "사진을 업로드하세요"
  │  
  │  최근 분석 섹션:
  │  ┌── 글래스 카드 리스트 ──┐
  │  │ [썸네일] 캐주얼 룩  78% │
  │  │ [썸네일] 미니멀 룩  92% │
  │  └────────────────────────┘
  │
  │  입력 경로:
  │  ├── A. 업로드 영역 탭 → 갤러리 선택
  │  └── B. 외부 앱 → Share Extension
  │
  ▼ (이미지 선택 후)
  AI 분석 중 (로딩)
  │  [Claude Haiku API 1회 호출 → JSON 응답]
  │  [매칭 엔진: DB 쿼리 + 점수 계산]
  │
  ▼ (~3-5초)
#10. 매칭 결과
  │  헤더: ← 뒤로 + "결과" + 공유 아이콘
  │  
  │  ┌── 매칭 점수 영역 ─────────────┐
  │  │ "78"  (Lobster 64px, primary)  │
  │  │ "Match Score"                  │
  │  └───────────────────────────────┘
  │  
  │  매칭 성공 아이템:
  │  ┌── 글래스 카드 ──────────────────┐
  │  │ [✓ 초록] 아이보리 니트     92%  │
  │  │          상의 · 색상 유사       │
  │  ├──────────────────────────────── ┤
  │  │ [✓ 초록] 카키 슬랙스      71%  │
  │  │          하의 · 스타일 일치     │
  │  └─────────────────────────────── ┘
  │  
  │  갭 아이템:
  │  ┌── 글래스 카드 ──────────────────┐
  │  │ [✗ 빨강] 브라운 로퍼           │
  │  │          이 아이템이 있으면     │
  │  │          완벽해요!     [찾기 →] │
  │  └─────────────────────────────── ┘
  │
  │  [찾기] 탭 → 딥링크 (무신사/에이블리/지그재그)
  │
  └── 하단: [이미지 저장] [공유하기] 버튼
```

**핵심 UX 포인트:**
- 매칭 점수는 **Lobster 64px** 대형 표시 → 첫눈에 결과 인지
- 매칭/갭 아이템은 **글래스 카드 리스트** 패턴 사용
- 갭 아이템은 부정적이지 않게 → **"있으면 완벽해요!"** 긍정 표현
- [찾기] 버튼은 **인디고 텍스트 + 외부 링크 아이콘**

### 4.6 인터랙션 & 모션

| 인터랙션 | 동작 | 용도 |
|----------|------|------|
| Pull-to-refresh | 스프링 바운스 | 홈, 옷장 |
| Long press | 300ms 후 다중 선택 모드 | 옷장 아이템 |
| Tab switch | Crossfade 300ms | 탭바 전환 |
| 로딩 | 단계별 체크 애니메이션 | 룩 매칭 분석 중 |
| 성공 피드백 | Confetti + Haptic | 매칭 결과, 아이템 등록 |
| 바텀시트 | Slide up 300ms ease-out | 프리미엄 모달 |

### 4.7 접근성

- 색상 대비 WCAG AA 이상 (4.5:1)
- 동적 텍스트 크기 지원 (Dynamic Type / Scalable Text)
- VoiceOver / TalkBack 대응 (의미 있는 접근성 라벨)
- 색약 대응: 매칭 성공 = 초록 + ✓ 텍스트, 실패 = 빨강 + ✗ 텍스트
