## 1. 개요

### 1.1 문제 정의

사람들은 인스타그램, 핀터레스트 등에서 인플루언서 코디를 참고하며 옷을 계속 구매하지만, "옷은 많은데 입을 옷이 없다"는 만성적 불만이 반복된다. **영감을 얻는 행위 자체가 소비를 유발하는 구조**에 갇혀 있다.

### 1.2 솔루션

AURA는 현재 보유한 옷을 촬영하면 AI가 코디를 추천하고, 인플루언서 룩을 **내 옷으로 재현하는 방법**을 제시하는 모바일 앱이다. 새 옷 구매 없이 스타일링 만족감을 제공한다.

### 1.3 가치 제안

"지출 절감"보다 **"나만의 스타일 발견"**으로 포지셔닝한다. 인플루언서 룩 재현 기능이 핵심 차별점이다.

### 1.4 타겟 사용자

| 구분 | 세그먼트 A (성장 엔진) | 세그먼트 B (매출 엔진) |
|------|----------------------|----------------------|
| 페르소나 | 인스타 탐색탭 중독자 | 출근 코디 고민러 |
| 연령 | 20대 초중반 여성 | 20대 후반~30대 초반 직장 여성 |
| 페인 | 인플 코디 따라 사면 안 어울림, 비슷한 옷 또 삼 | 아침마다 옷 앞에서 15분 낭비, 조합을 모름 |
| 반응 포인트 | 룩 재현 기능에 강하게 반응 | 매일 코디 추천으로 시간 절약 |
| 역할 | 사용자 확보 + 바이럴 | 매출 + 리텐션 |
| 리스크 | 구독 결제 민감, 전환율 낮을 수 있음 | 지불 의향 가장 높음 |

**실행 전략:** 마케팅은 A에 집중 (틱톡/릴스 바이럴), 프로덕트는 B도 만족 (매일 코디 추천)

## 2. 경쟁 환경

### 2.1 주요 경쟁사

| 경쟁사 | 국가 / 유저 | 강점 | 약점 |
|--------|-----------|------|------|
| Whering | 영국, 900만+ | 소셜 기능 중심 | AI 추천 베타 수준, 한국어 미지원 |
| Cladwell | 미국, 100만+ 다운로드 | 캡슐 워드로브 철학 | AI 품질 낮음, 한국어 미지원 |
| Acloset | 한국, 400만 유저 | AI 패션 비서 | 무료 100개 아이템까지만, 스타일 추천 부족 |

### 2.2 공통 한계점 (= AURA의 기회)

1. **옷 등록 허들이 너무 높음** — 하나씩 사진 찍고 태그, 대부분 중간 포기
2. **AI 코디 추천 품질 낮음** — 실제로 쓸 만한 수준 아님
3. **"소비 줄이기" 표방하지만 결국 쇼핑 유도** — 수익 구조가 쇼핑에 의존
4. **한국 시장 맞춤 서비스 없음** — 무신사/에이블리/지그재그 연동 없음

## 3. 차별화 전략

### 3.1 "오늘 입은 옷 1벌부터 시작" 온보딩

기존 앱들이 "옷장을 먼저 채우세요"라고 요구하지만, 이것이 가장 큰 이탈 요인이다.

**AURA 온보딩 전략:**
- 가입 직후 → **오늘 입은 옷 전신 사진 1장** 촬영
- 30초 만에 → 배경 제거 + 색상 추출 + 카테고리 탭 선택 → **3개 아이템 자동 등록**
- 즉시 → 핵심 경험 제공 (기본 코디 추천 또는 룩 재현)

**AURA 온보딩 UX (v4.9):**
- 스플래시 A (인디고 그라디언트, Lobster 로고) → 스플래시 B (소프트 그라디언트) → 온보딩 3장 (중앙 정렬, 동심원 아이콘, 인디고 배경) → 로그인 (소프트 배경, 카카오 노란색 + Apple 블랙) → Home 탭 진입
- 온보딩 2~3번째 화면에 뒤로가기 지원, 전체 건너뛰기 가능

**갤러리 스캔을 초기 전략에서 제외한 이유:**
- 다인 사진에서 사용자 구분 불가
- 사진 없는 사용자 대응 불가
- 프라이버시 거부감

> 철학: **"옷장 먼저 채우고 시작"이 아닌 "핵심 경험 먼저 주고 옷장은 쓰면서 채운다"**

### 3.2 "내 옷으로 인플루언서 룩 재현" (핵심 차별점)

경쟁사 어디에도 없는 기능. 서비스의 핵심 Hook이자 바이럴 포인트.

### 3.3 한국 패션 생태계 최적화

무신사/에이블리/지그재그/29CM 연동, 퍼스널 컬러 시스템 반영

### 3.4 수익 모델 — "쇼핑"이 아닌 "스타일링 구독"

경쟁사들이 결국 쇼핑 유도로 수익을 내는 반면, AURA는 **"꼭 필요한 1벌만 추천"** 방식으로 신뢰를 쌓는다.

### 4.2 Tier 1: MVP 핵심 기능 (0~3개월)

#### F1. 온보딩 — 오늘 입은 옷 1벌 촬영

**목적:** 가입 후 30초 안에 핵심 가치를 경험하게 한다.

**사용자 흐름:**
```
가입 완료
  → "오늘 입은 옷을 찍어보세요!" 프롬프트
  → 기본 카메라 앱으로 전신 사진 촬영 (자체 카메라 UI 개발 안 함)
  → 자동 처리:
      1) 배경 제거 (remove.bg / rembg)
      2) 아이템 분리 (상의/하의/아우터/신발 감지)
      3) 각 아이템별 색상 자동 추출 (K-Means)
      4) 카테고리 사용자 탭 선택 (AI가 아닌 사용자가 탭으로 확정)
  → 옷장에 3개 아이템 등록 완료
  → "룩 재현해볼까요?" 또는 기본 코디 추천 제시
```

**상세 요구사항:**

| 항목 | 요구사항 |
|------|---------|
| 카메라 | 기본 OS 카메라 앱 호출 (자체 카메라 UI 없음) |
| 배경 제거 | remove.bg API Free (월 50회) → rembg fallback |
| 아이템 분리 | 전신 사진에서 상의/하의/아우터/신발 영역 분리 |
| 색상 추출 | K-Means 클러스터링 기반 dominant color 추출 |
| 카테고리 선택 | 사용자가 탭 UI로 직접 선택 (상의/하의/아우터/원피스/신발/가방/액세서리) |
| AI 호출 | 이 기능에서는 **0회** (비용 0원) |
| 목표 시간 | 촬영부터 등록 완료까지 30초 이내 |

**카테고리 분류 체계:**

```
Top Category (대분류)
├── 상의 (Tops)
│   ├── 티셔츠 / 셔츠 / 블라우스 / 니트 / 맨투맨 / 후디 / 조끼
├── 하의 (Bottoms)
│   ├── 청바지 / 슬랙스 / 반바지 / 치마 / 레깅스
├── 아우터 (Outerwear)
│   ├── 자켓 / 코트 / 패딩 / 가디건 / 바람막이
├── 원피스 (Dresses)
│   ├── 미니 / 미디 / 맥시 / 점프수트
├── 신발 (Shoes)
│   ├── 스니커즈 / 부츠 / 샌들 / 로퍼 / 힐
├── 가방 (Bags)
│   ├── 백팩 / 숄더백 / 크로스백 / 토트백 / 클러치
└── 액세서리 (Accessories)
    ├── 모자 / 스카프 / 벨트 / 주얼리 / 선글라스
```

**아이템 속성 (각 아이템에 저장):**

| 속성 | 입력 방식 | 필수 여부 |
|------|----------|----------|
| 카테고리 (대/소) | 사용자 탭 선택 | 필수 |
| 색상 (dominant color hex) | 자동 추출 | 필수 |
| 색상명 (한국어) | hex → 근사 색상명 매핑 | 자동 |
| 이미지 (배경 제거된) | 자동 처리 | 필수 |
| 원본 이미지 | 자동 저장 | 필수 |
| 스타일 태그 | 사용자 선택 (캐주얼/포멀/스트릿/미니멀/빈티지 등) | 선택 |
| 브랜드 | 사용자 입력 | 선택 |
| 시즌 | 사용자 선택 (봄/여름/가을/겨울/사계절) | 선택 |
| 등록일 | 자동 | 자동 |
| 마지막 착용일 | 자동 (데일리 기록 시 업데이트) | 자동 |

#### F2. 인플루언서 룩 재현 (핵심 차별 기능)

**목적:** 인플루언서/레퍼런스 이미지의 코디를 내 옷장으로 재현한다.

**AURA 앱 내 위치 (v4.9):** Match 탭 (3번째 탭, 영문 타이틀 "Match" Lobster)

**사용자 흐름:**
```
Match 탭 진입 → 매칭 업로드 화면 (화면 #9)
  레퍼런스 이미지 입력 (2가지 경로)
    경로 A: 앱 내에서 갤러리 이미지 선택
    경로 B: 다른 앱(인스타 등)에서 공유 버튼으로 전달
  
  → AI 분석 (Claude Haiku API, 1회 호출)
      - 이미지 내 패션 아이템 감지
      - 각 아이템 속성 추출:
        · 카테고리 (상의/하의/아우터/신발/가방/액세서리)
        · 색상 (hex 코드 + 색상명)
        · 스타일 키워드 (캐주얼, 포멀, 스트릿 등)
        · 핏/실루엣 (오버사이즈, 슬림, 레귤러 등)
        · 패턴 (무지, 스트라이프, 체크, 플로럴 등)
        · 소재 감지 (가능한 경우)
  
  → 내 옷장 매칭 (서버 코드 로직, AI 호출 아님)
      - 아이템별 유사도 점수 계산
      - 가장 높은 총합 점수 조합 선택
  
  → 매칭 결과 화면 (화면 #10)
      - 매칭 점수 (Lobster 64px, 인디고)
      - 매칭 성공 아이템: 글래스 카드, 초록 체크 아이콘
      - 매칭 실패 아이템: 글래스 카드, 빨간 X 아이콘 + "찾기" 버튼
  
  → 갭 분석
      - 매칭 실패 아이템 하이라이트 ("이 아이템이 있으면 완벽해요!")
      - 외부 검색 링크 제공 (딥링크 방식)
```

**상세 요구사항:**

| 항목 | 요구사항 |
|------|---------|
| 레퍼런스 입력 | 갤러리 선택 + 앱 간 공유(Share Extension) |
| AI 분석 | Claude Haiku API 1회 호출 (이미지 + 프롬프트 → JSON 응답) |
| 프롬프트 | 구조화된 JSON 응답 요청 (아이템 목록 + 속성) |
| 매칭 엔진 | 서버 코드 로직 (AI 추가 호출 없음) |
| 매칭 알고리즘 | 카테고리 일치 → 색상 유사도 → 스타일 일치 → 종합 점수 |
| 결과 표시 | 매칭 점수 (Lobster) + 아이템별 매칭/갭 리스트 (글래스 카드) |
| 갭 표시 | 매칭 실패 아이템 + "이 아이템이 있으면 완벽해요!" 라벨 |
| 외부 링크 | 딥링크 방식 (MVP에서 커미션 없음, 사용자 가치 제공) |
| 무료 제한 | 월 5회 |
| 프리미엄 | 무제한 |

**Claude Haiku API 호출 스펙:**

```
Input:
- 레퍼런스 이미지 (base64)
- 시스템 프롬프트: 패션 아이템 분석 지시
- 출력 포맷: JSON

Expected Output:
{
  "items": [
    {
      "category": "상의",
      "subcategory": "니트",
      "color": {
        "hex": "#2C3E50",
        "name": "네이비"
      },
      "style": ["캐주얼", "미니멀"],
      "fit": "오버사이즈",
      "pattern": "무지",
      "material": "울 혼방"
    },
    ...
  ],
  "overall_style": "캐주얼 미니멀",
  "occasion": "일상"
}
```

**매칭 점수 계산 (MVP):**

```
총점 = 카테고리 일치 (40점)
     + 색상 유사도 (30점)
     + 스타일 일치 (20점)
     + 핏/패턴 보너스 (10점)

색상 유사도 = 30 - (color_distance(hex1, hex2) / max_distance * 30)
  - color_distance: CIEDE2000 또는 단순 Euclidean (RGB)

스타일 일치 = (겹치는 키워드 수 / 전체 키워드 수) * 20

매칭 임계값: 총점 50점 이상 → "매칭 성공"
             총점 50점 미만 → "매칭 실패" → 갭 분석 대상
```

**MVP에서 제외 (Phase 2+):**
- 스타일 벡터 기반 분위기 매칭 (딥 메트릭 러닝, 512차원)
- 조합 호환성 검증 (아이템 간 어울림 판단)
- 아이템 스왑 기능 (특정 아이템만 교체)
- 체형/퍼스널 컬러 보정
- 코디 버전 다양화 (3가지 버전 제안)

| 기능 | 설명 |

## 5. 기술 아키텍처

### 5.1 설계 원칙

> **자체 AI 서버 운영 없이 API 호출만으로 구성한다.**
> 1인 개발자가 관리할 것은 "앱 하나"뿐이어야 한다.
> GPU 서버 세팅, 모델 배포, 스케일링, 장애 복구를 혼자 하는 것은 비현실적이다.

### 5.2 시스템 구성도

```
CLIENT (Flutter iOS/Android)
    │
    │ HTTPS (REST API)
    ▼
API LAYER (Supabase Edge Functions - Deno/TypeScript)
    │
    ├── Supabase PostgreSQL (DB + Auth + RLS)
    ├── Cloudflare R2 (이미지 저장 + CDN)
    └── Claude Haiku API (AI 추론, 룩 재현 전용)
```

### 5.3 기능별 기술 매핑

| 기능 | 기술 | AI 호출 | 비용 |
|------|------|---------|------|
| 사진 촬영 | OS 기본 카메라 호출 | 없음 | 0원 |
| 배경 제거 | remove.bg API Free → rembg | 없음 | 0원 |
| 색상 추출 | K-Means 클러스터링 | 없음 | 0원 |
| 카테고리 선택 | 사용자 탭 UI | 없음 | 0원 |
| 룩 재현 (레퍼런스 분석) | Claude Haiku API | **1회/건** | ~$0.001/건 |
| 룩 재현 (옷장 매칭) | DB 쿼리 + 점수 계산 코드 | 없음 | 0원 |
| 기본 코디 추천 | 규칙 기반 코드 로직 | 없음 | 0원 |
| 갭 분석 외부 링크 | 딥링크 URL 생성 | 없음 | 0원 |

### 5.4 인프라 스택

| 구성 요소 | 선택 | 비고 |
|-----------|------|------|
| 모바일 앱 | Flutter (Dart) | iOS/Android 동시 개발, 1인 개발자 최적 |
| API 서버 | Supabase Edge Functions (Deno/TS) | 서버 관리 불필요, 자동 스케일링 |
| 데이터베이스 | Supabase PostgreSQL | RLS 내장, Free → Pro |
| 이미지 저장소 | Cloudflare R2 | S3 호환, 무료 10GB |
| AI 추론 | Claude Haiku API (종량제) | 건당 ~$0.001 |
| 인증 | Supabase Auth | 카카오/Apple 소셜 로그인 |
| 푸시 알림 | Firebase Cloud Messaging | 무료 |
| 날씨 데이터 | OpenWeather API 등 | 무료 티어 |
| 모니터링 | Sentry (Free tier) | 에러 추적 |
| 자체 GPU 서버 | **없음** | API로 대체 |
